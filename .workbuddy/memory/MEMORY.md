## 四、项目开发规则（2026-04-08 确定）

### 开发规范
1. **积分暂停机制**：积分消耗超过2000时暂停，等待用户确认继续
2. **三检制度**：每模块完成后执行代码自检 → 复检 → 最终行为测试
3. **自动继续**：开发过程中卡住时尝试继续执行
4. **版本管理**：每模块完成后记忆储存，并上传GitHub min分支

### Git分支策略
- main分支：最终稳定版本
- dev分支：开发中版本
- min分支：每模块完成后上传存档

### 开发流程
每模块开发遵循：
1. 需求确认
2. 代码编写
3. 自检（语法、逻辑）
4. 复检（功能完整性）
5. 行为测试（实际运行测试）
6. 记忆更新
7. Git提交
